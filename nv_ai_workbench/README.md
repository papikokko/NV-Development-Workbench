# NV AI Workbench

## 目的

このワークベンチは、NV開発における市場・商品性・顧客体験・NV要求・設計部位・評価結果・判断・知見化を、AIで一気通貫につなぐためのモック環境である。

現行業務プロセスをそのまま再現することが目的ではない。NV開発に必要な判断要素を抽出し、AIがそれらをつなぐとどのような開発支援が可能になるかを実演する。

## 重要思想

AIが判断を出すだけではなく、**判断に使った入力情報・判断軸・根拠をトレースとして残す**。

## 基本導線

```
市場・ユーザートレンド
  ↓
商品性
  ↓
顧客体験シーン
  ↓
NV要求
  ↓
NV現象・音振特性
  ↓
車両システム・設計部位
  ↓
評価計画
  ↓
評価結果・官能コメント
  ↓
商品性未達リスク判定
  ↓
原因仮説・追加検証
  ↓
DRBFM・判断メモ・知見化
  ↓
判断トレース・Evidence Map として根拠を残す
```

## フォルダ構成

| フォルダ | 役割 | シナリオ差し替え時 |
|---------|------|------------------|
| `common/` | NV現象、車両システム、解析手法、判断ルール、出力テンプレートを格納 | 使い回す |
| `scenarios/` | 個別車両、商品性、評価条件、仮データを格納 | 差し替える |
| `workbench/` | AIの実行手順、プロンプト、実行ログを格納 | 使い回す |
| `scenarios/{name}/05_output/` | 各シナリオの成果物と判断トレース | 毎回生成する |

## シナリオの考え方

- `common/` はシナリオ間で共有する辞書・判断ルール
- `scenarios/` 配下に車種・商品コンセプト単位でフォルダを切る
- 新シナリオは `scenarios/scenario_002_...` として追加する

## 必須出力（すべてのシナリオで共通）

| ファイル | 役割 |
|---------|------|
| `decision_trace.csv` | 判断ごとに使用入力・データID・判断軸・根拠・確からしさを一覧化 |
| `evidence_map.md` | 評価結果・官能コメント・設計変更・原因仮説のつながりを文章で説明 |
| `input_usage_log.md` | 各StepでAIがどのファイル・項目を使ったか記録 |
