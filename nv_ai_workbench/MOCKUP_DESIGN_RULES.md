# NVワークベンチ モック設計ルール & ノウハウ

作成: 2026-05-25  
対象: `nv_workbench_mockup_*.html` 系列の設計・改修時に参照すること

---

## 1. シナリオ設計の大原則

### 1-1. 時系列整合性（最重要）

各Stepに登場するデータ・数値・判断は、**その時点で「存在しうる情報か」を必ず確認すること。**

| Step | 存在しうる情報 | 存在してはいけない情報 |
|------|--------------|-------------------|
| Step 0（キックオフ） | 競合現状値・前機種実績値・市場トレンド | **E:NX2目標値**（まだ決まっていない） |
| Step 1（競合分析） | 競合スコア・前機種スコア・市場トレンド | 自社の開発目標値 |
| Step 2（コンセプト選択） | ← **ここで初めて目標方向が決まる** | |
| Step 3以降 | 目標値・要求値（Step 2で決まったもの） | ✓ OK |

**NG例（実際にやってしまった失敗）:**
- Step 0の散布図に「Honda E:NX2（目標）」の点を表示していた → 削除
- Step 0のブレイクダウンに「E:NX2目標バー」を表示していた → 削除

### 1-2. V字フローの閉じ方

Step 6（DR向け整理）は必ず **Step 0の全体商品性観点に戻って成立性を確認する** こと。  
"V字の起点から成立が確認できた" という一文を最後に入れることで流れが完結する。

ただし、Step 0で削除したデータを参照しないよう注意（例：Step 0に目標値がなければ、Step 6で「Step 0で示した目標値と一致」とは書けない）。

### 1-3. コンセプト・目標・要求の登場順

```
Step 0: 競合立ち位置（目標なし）
Step 1: NV7軸ベンチマーク（目標なし、前機種実績のみ）
Step 2: コンセプト選択 ← 目標方向（例：IONIQ5水準8.0+）が初めて登場
Step 3: NV要求数値（R01〜R08）← 数値目標はここで確定
Step 4以降: 目標値との比較・照合
```

## 2. 文章・ナレーションのルール

### 2-1. ですます調に完全統一

全てのナレーションボックス（`background:#f0f4f8` の帯状説明文）は **ですます調** に統一する。

**NG：**
- 「この段階でリスクを予測し、対策の準備を先行させることが目的だ。」
- 「測定値と官能の両方で確認された課題は、顧客が実際に感じている問題だ。」
- 「→ これらを対象に、Step 5で対策の優先順位を決める。」

**OK：**
- 「この段階でリスクを予測し、対策の準備を先行させることが目的です。」
- 「→ Step 5で対策の優先順位を決めます。」

**チェック方法：** HTMLをGrep(`だ。|である。|だろう。`)で検索して残存確認すること。

### 2-2. 目的ファースト構造

各ナレーションは以下の順序で書く：

1. **何のために（目的）** — なぜこの分析をやるのか
2. **何をして（方法）** — どんな手法・比較をしているか
3. **次何をするか（接続）** — ボールドの `→ 次の〜` で明示

**NG（やったことを羅列している）：**
「後席NLは6.9点、IONIQは8.1点、差は1.2ptです。風切り音は...」

**OK（目的から入る）：**
「市場トレンドで競合の静粛性レベルが急上昇していると確認しました。このチャートは、具体的に誰が・どの軸で・どこまで上げてきたかを確認するためのものです。〜 **→ 次のStepでは...を決めます。**」

### 2-3. AIの初期発言

各Stepに入った時のAI最初の発言は、**状況設定から入る**。前置きや自己紹介なし。

**NG：** 「NVワークベンチへようこそ。これからStep 1の分析を始めます。」  
**OK：** 「キックオフ直後です。まず競合4社の立ち位置を全体商品性マップで確認します。」

## 3. データ・ビジュアライズのルール

### 3-1. 前機種は「前機種実績」、青色は一意に使う

ベンチマーク系チャートで青色（`#4a9eff`）は **Honda E:NX1（前機種実績）** のみに割り当てる。  
設計目標・E:NX2目標に青を使わない（混同の元になる）。

**チャート凡例で必ずチェックすること：**
- 青線/青バー = 前機種実績 ✓
- 橙線 = 設計変更込み予測値 ✓
- 灰線 = ベースライン ✓

### 3-2. スコアスケール説明

1〜10スコアを初めて使うチャートの直前に必ず説明を入れる：

```html
<div style="...background:#f8fafc;font-size:10px;color:var(--t2)">
  スコアは1〜10の推定値（高いほど静粛性が優れている）。各社エンジニアの官能評価と第三者テスト機関の客観測定を組み合わせて変換。
</div>
```

### 3-3. データの二重性（意図的）

Step 1 と Step 4 で Honda E:NX の数値が異なるのは **意図的な設計**。

| Step | 値 | 意味 |
|------|---|------|
| Step 1（競合比較） | E:NX1前機種実績 | 開発出発点の比較基準 |
| Step 4（実測照合） | E:NX2測定値（目標未達） | 設計変更の実際の影響 |

この差（NG）がデモのストーリーコア。設計変更DC01+DC02の複合影響で目標を下回った → 再設計判断へ。

### 3-4. ゲージバーの3ライン

Step 4の測定結果ゲージバーには3本のライン：
- 🔵 **目標ライン**（data-v付きブルーライン）
- 🟠 **最低許容ライン**（オレンジ）
- 🟢 **魅力ライン**（グリーン）

ラインなしで棒グラフだけは不完全。

## 4. シナリオ整合チェックリスト

モックを作った・改修した後は以下を必ず確認すること：

### Step 0
- [ ] 競合の現状値のみ表示（自社目標なし）
- [ ] 散布図に「E:NX2目標」点がない
- [ ] ブレイクダウン棒グラフに「E:NX2目標バー」がない
- [ ] 「なぜNVドメインに集中するか」の根拠が数値で示されている

### Step 1
- [ ] 前機種実績（E:NX1）が青線で表示
- [ ] スコア説明（1〜10）が表の上にある
- [ ] NG（赤）/ OK（緑）の色のみ意味を持つ。青バックに意味を持たせない
- [ ] 「→ 次のStepでは〜を決めます」で接続

### Step 2
- [ ] ここで初めて「目標方向」が決まる（数値確定はStep 3）
- [ ] 3案比較で「なぜ案Aか」の根拠が市場トレンド・商品性で説明されている

### Step 3
- [ ] 設計変更リスク予測は実測前の段階として記述
- [ ] アーキテクチャーチャートの目標値ラインはStep 2確定後の値
- [ ] サロゲートモデルの「目的」が3層（内音/外音/全性能）で明示されている
- [ ] Pareto Frontの意味（トレードオフ可視化）が説明されている

### Step 4
- [ ] 実測値 vs 目標値（Step 3確定値）の比較
- [ ] 官能コメントと測定値が照合されている（CMT × RSLT対応）
- [ ] SVG振動伝達経路図でSource/Path/Receiverが色分け済み

### Step 5
- [ ] 課題ランキングが「dBの大きさ順」でなく「商品性影響の大きさ順」
- [ ] BO候補マップでexploit/explore/mixedが区別されている
- [ ] 「なぜこもりが1位か」がスコア差（数値）で説明できる

### Step 6
- [ ] Step 0のギャップ分析に戻って成立性を確認する文がある
- [ ] ただし Step 0 に存在しないデータを参照していない
- [ ] DRBFM/判断メモ/知見カードの3点セット
- [ ] 商品性副作用テーブルでNV/電費/重量/コスト/操安を全項目確認

## 5. UI設計のルール

### 5-1. Stepナビゲーション
- 各Stepは独立パネル。自動再生ではなくStepごとに進む設計。
- 各Stepの先頭に「状況設定バナー（ctx-banner）」で「今なぜこれをやるか」を明示。
- `ctx-q`（黄色バッジ）でそのStepの**判断ポイント**を1文で示す。

### 5-2. インプットバッジ（step-io）
各Stepの状況設定バナー直下に、参照するデータファイルと出力判断を表示する：

```html
<div class="step-io">
  <span class="io-label">📂 参照データ:</span>
  <span class="io-file">ファイル名.csv</span>
  <span class="io-arrow">→</span>
  <span class="io-out">このStepでの出力・判断内容</span>
</div>
```

### 5-3. チャート目的バー（cpb）
ビジュアライズの直前に必ず「なぜこのチャートを見るか」を`cpb`クラスで表示。

### 5-4. デモモード
- **自動再生**: 全Step通し（30秒程度）
- **Step別**: 1Stepずつ手動確認
- **Real AI**: ローディング → 思考プロセス → 回答のフロー
